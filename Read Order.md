# Read Order & Architecture Guide - Starter IL

## 1. ארכיטקטורת המערכת (High-Level Architecture)
המערכת נבנתה כ-Single Page Application (SPA) מודרנית באמצעות **React, Vite ו-Tailwind CSS** (חוללה וקודדה בעזרת Google AI Studio / Gemini). 

בשלב ה-MVP הנוכחי, אנו מיישמים ארכיטקטורת **Mock Backend** (ללא שרת אמיתי). אנו משתמשים ב-`React Context API` בשילוב `localStorage` כדי לסמלץ מסד נתונים חי (ניהול מלאי, הזמנות, וסשן משתמש). 
המטרה היא להוכיח לוגיקה עסקית (Business Logic), מניעת התנגשויות (Race Conditions) ויציבות Front-End מוחלטת, רגע לפני החיבור למסד נתונים אמיתי (כגון Supabase) בשלבים הבאים.

## 2. סדר קריאת הקוד המומלץ והיכן מסתתרת הלוגיקה
כדי להבין את זרימת המידע במערכת ואת הטיפול במקרי הקצה, אנו ממליצים לעבור על הקבצים לפי הסדר הבא:

### א. `src/context/AppContext.jsx` (מנוע הנתונים וה-Mock DB)
*   זהו הלב של המערכת, בו מנוהל ה-State הגלובלי (`products`, `cart`, `orders`, `user session`).
*   **מה לחפש כאן?** את ה-`initialProducts` (Mock Data). שימו לב ששדה ה-`category` הוגדר כטקסט חופשי (Dynamic Category) ולא כ-Enum קשיח, כדי להעניק לאדמין גמישות עתידית להוספת קטגוריות ללא תלות במתכנת.

### ב. `src/pages/Checkout.jsx` (קופה וולידציות)
*   כאן מתבצעת הרכישה המדומה, ופה החבאנו את מנגנוני ההגנה המרכזיים של המערכת:
*   **מה לחפש כאן?** 
    1. **Double Cart Validation (מניעת Race Condition):** לפני אישור ההזמנה, רצה פונקציית `validateCartBeforeCheckout` אשר מוודאת מול ה-Context הגלובלי שהמלאי לא השתנה בזמן שהלקוח שהה בסל. אם המלאי ירד, הכמות בעגלה מתעדכנת אוטומטית והלקוח מקבל התראה.
    2. **Strict Client-Side Validation:** טופס ה-Checkout עטוף בוולידציות HTML5 קשוחות (שדות חובה, Regex לטלפון ישראלי מתחיל ב-05) כדי למנוע הזנת נתוני זבל למערכת.

### ג. `src/pages/AdminDashboard.jsx` (פאנל ניהול והחזרות)
*   אזור הניהול נגיש רק כאשר `role === 'admin'`.
*   **מה לחפש כאן?** את פונקציית `handleOfflineRefund`. מכיוון שהחלטנו שביטולים יתבצעו רק דרך מוקד טלפוני (כדי למנוע פחת של מוצרים פיזיים ושבירים), הפונקציה הזו מדגימה כיצד שינוי סטטוס הזמנה ל-`Refunded_Offline` על ידי האדמין **מחזיר אוטומטית את הפריטים למלאי הזמין** (`stock`).

### ד. `src/components/ProductCard.jsx` & `src/pages/Home.jsx` (הצגת מלאי)
*   אלו רכיבי התצוגה של הקטלוג.
*   **מה לחפש כאן?** את הלוגיקה הוויזואלית של המלאי. כשהמלאי יורד ל-4 ומטה מופיע באנר התראה כתום. כש-`stock === 0` התמונה מקבלת פילטר Grayscale, כפתור ההוספה מנוטרל (Disabled), ומופיע תג "אזל במלאי".

### ה. `src/pages/QRInfoPage.jsx` (ה-Viral Loop)
*   ראוט ציבורי נטול הרשאות (`/kit/:id`) המיועד לסריקה מברקוד שעל הקופסה הפיזית.
*   **מה לחפש כאן?** את המיקוד בשירות עצמי: צ'קליסט תכולה, כפתור מעבר מהיר לוואטסאפ (לטיפול בחוסרים/שברים) ובאנר שיווקי המייצר Viral Loop ומפנה לרכישה באתר עבור משתמשים שסרקו את הערכה אצל חברים.
